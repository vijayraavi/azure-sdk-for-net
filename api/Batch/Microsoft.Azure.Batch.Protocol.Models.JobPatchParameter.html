OnAllTasksComplete

Gets or sets the action the Batch service should take when all tasks in the job are in the completed state.

Declaration
public OnAllTasksComplete? OnAllTasksComplete { get; set; }
Property Value
Type Description
System.Nullable<OnAllTasksComplete>
Remarks

If omitted, the completion behavior is left unchanged. You may not change the value from terminatejob to noaction - that is, once you have engaged automatic job termination, you cannot turn it off again. If you try to do this, the request fails with an 'invalid property value' error response; if you are calling the REST API directly, the HTTP status code is 400 (Bad Request). Possible values include: 'noAction', 'terminateJob'

PoolInfo

Gets or sets the pool on which the Batch service runs the job's tasks.

Declaration
public PoolInformation PoolInfo { get; set; }
Property Value
Type Description
PoolInformation
Remarks

You may change the pool for a job only when the job is disabled. The Patch Job call will fail if you include the poolInfo element and the job is not disabled. If you specify an autoPoolSpecification specification in the poolInfo, only the keepAlive property can be updated, and then only if the auto pool has a poolLifetimeOption of job. If omitted, the job continues to run on its current pool.
