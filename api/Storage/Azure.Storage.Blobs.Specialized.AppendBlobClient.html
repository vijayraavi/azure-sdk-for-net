Class AppendBlobClient

The AppendBlobClient allows you to manipulate Azure Storage append blobs.

An append blob is comprised of blocks and is optimized for append operations. When you modify an append blob, blocks are added to the end of the blob only, via the AppendBlockAsync(Stream, Byte[], Nullable<AppendBlobAccessConditions>, IProgress<StorageProgress>, CancellationToken) operation. Updating or deleting of existing blocks is not supported. Unlike a block blob, an append blob does not expose its block IDs.

Each block in an append blob can be a different size, up to a maximum of 4 MB, and an append blob can include up to 50,000 blocks. The maximum size of an append blob is therefore slightly more than 195 GB (4 MB X 50,000 blocks).

AppendBlockAsync(Stream, Byte[], Nullable<AppendBlobAccessConditions>, IProgress<StorageProgress>, CancellationToken)

The AppendBlockAsync(Stream, Byte[], Nullable<AppendBlobAccessConditions>, IProgress<StorageProgress>, CancellationToken) operation commits a new block of data, represented by the content , to the end of the existing append blob. The AppendBlockAsync(Stream, Byte[], Nullable<AppendBlobAccessConditions>, IProgress<StorageProgress>, CancellationToken) operation is only permitted if the blob was created as an append blob.

For more information, see https://docs.microsoft.com/rest/api/storageservices/append-block.

CreateAsync(Nullable<BlobHttpHeaders>, System.Collections.Generic.IDictionary<String, String>, Nullable<AppendBlobAccessConditions>, CancellationToken)

The CreateAsync(Nullable<BlobHttpHeaders>, System.Collections.Generic.IDictionary<String, String>, Nullable<AppendBlobAccessConditions>, CancellationToken) operation creates a new 0-length append blob. The content of any existing blob is overwritten with the newly initialized append blob. To add content to the append blob, call the AppendBlockAsync(Stream, Byte[], Nullable<AppendBlobAccessConditions>, IProgress<StorageProgress>, CancellationToken) operation.

For more information, see https://docs.microsoft.com/rest/api/storageservices/put-blob.
