ForceCloseHandles(String, String, CancellationToken)

The ForceCloseHandles(String, String, CancellationToken) operation closes a handle or handles opened on a file at the service. It supports closing a single handle specified by handleId or or closing all handles opened on that resource.

This API is intended to be used alongside GetHandlesAsync(CancellationToken) to force close handles that block operations. These handles may have leaked or been lost track of by SMB clients. The API has client-side impact on the handle being closed, including user visible errors due to failed attempts to read or write files. This API is not intended for use as a replacement or alternative for SMB close.

For more information, see .

Declaration
public virtual Response<StorageClosedHandlesSegment> ForceCloseHandles(string handleId = null, string marker = null, CancellationToken cancellationToken = null)
Parameters
Type Name Description
System.String handleId

Optional. Specifies the handle ID to be closed. If not specified, or if equal to "*", will close all handles.
System.String marker

An optional string value that identifies the segment of the handles to be closed with the next call to ForceCloseHandles(String, String, CancellationToken). The operation returns a non-empty if the operation did not return all items remaining to be closed with the current segment. The NextMarker value can be used as the value for the marker parameter in a subsequent call to request the closure of the next segment of handles.
CancellationToken cancellationToken

Optional to propagate notifications that the operation should be cancelled.
Returns
Type Description
Response<StorageClosedHandlesSegment>

A describing a segment of the handles closed.

Specifies the range of bytes to be written. Both the start and end of the range must be specified.
Stream content

A containing the content of the range to upload.
System.Byte[] transactionalContentHash

Optional MD5 hash of the range content. Must not be used when writeType is set to Clear.
